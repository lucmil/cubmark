Detailed Description

These methods are called every time a parameter is about to change. They are declared virtual to allow subclasses to act on those changes for special purposes. For example, to use C as the upper boundary values instead of scaling, like the G1 test function.

If you override any of these, remember to forward the call back to it's superclass or the vectors won't be updated (unless you do not want them to change).
